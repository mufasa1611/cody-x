Cody CLI po defaultu pokreće [TUI](/docs/tui) kada se pokrene bez ikakvih argumenata.

```bash
cody
```

Ali takođe prihvata naredbe kao što je dokumentovano na ovoj stranici. Ovo vam omogućava programsku interakciju sa Cody.

```bash
cody run "Explain how closures work in JavaScript"
```

---

### tui

Pokrenite Cody terminalski korisnički interfejs.

```bash
cody [project]
```

#### Opcije

| Opcija                                   | Kratko | Opis                                                                     |
| ---------------------------------------- | ------ | ------------------------------------------------------------------------ |
| <nobr><code>{"--continue"}</code></nobr> | `-c`   | Nastavite posljednju sesiju                                              |
| <nobr><code>{"--session"}</code></nobr>  | `-s`   | ID sesije za nastavak                                                    |
| <nobr><code>{"--fork"}</code></nobr>     |        | Forkujte sesiju pri nastavku (koristiti sa `--continue` ili `--session`) |
| <nobr><code>{"--prompt"}</code></nobr>   |        | Prompt za upotrebu                                                       |
| <nobr><code>{"--model"}</code></nobr>    | `-m`   | Model za korištenje u obliku provider/model                              |
| <nobr><code>{"--agent"}</code></nobr>    |        | Agent za korištenje                                                      |
| <nobr><code>{"--port"}</code></nobr>     |        | Port na kojem treba slušati                                              |
| <nobr><code>{"--hostname"}</code></nobr> |        | Hostname na kojem treba slušati                                          |

## Naredbe

Cody CLI takođe ima sljedeće naredbe.

---

### agent

Upravljajte Cody agentima.

```bash
cody agent [command]
```

---

### attach

Priključite terminal na već pokrenut Cody backend server pokrenut putem `serve` ili `web` naredbi.

```bash
cody attach [url]
```

Ovo omogućava korištenje TUI-ja sa udaljenim Cody backend-om. Na primjer:

```bash
# Start the backend server for web/mobile access
cody web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
cody attach http://10.20.30.40:4096
```

#### Opcije

| Opcija                                   | Kratko | Opis                                                                                          |
| ---------------------------------------- | ------ | --------------------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |        | Radni direktorij za pokretanje TUI-a                                                          |
| <nobr><code>{"--continue"}</code></nobr> | `-c`   | Nastavi posljednju sesiju                                                                     |
| <nobr><code>{"--session"}</code></nobr>  | `-s`   | ID sesije za nastavak                                                                         |
| <nobr><code>{"--fork"}</code></nobr>     |        | Forkuj sesiju prilikom nastavka (koristite sa `--continue` ili `--session`)                   |
| <nobr><code>{"--password"}</code></nobr> | `-p`   | Lozinka za osnovnu autentifikaciju (zadano: `CODY_SERVER_PASSWORD`)                       |
| <nobr><code>{"--username"}</code></nobr> | `-u`   | Korisničko ime za osnovnu autentifikaciju (zadano: `CODY_SERVER_USERNAME` ili `cody`) |

### auth

Naredba za upravljanje vjerodajnicama i prijavom za provajdere.

```bash
cody auth [command]
```

---

#### login

Cody pokreće lista provajdera na [Models.dev](https://models.dev), tako da možete koristiti `cody auth login` da konfigurirate API ključeve za bilo kojeg provajdera kojeg želite koristiti. Ovo je pohranjeno u `~/.local/share/cody/auth.json`.

```bash
cody auth login
```

Kada se Cody pokrene, učitava dobavljače iz datoteke vjerodajnica. I ako postoje neki ključevi definirani u vašim okruženjima ili `.env` fajl u vašem projektu.

---

#### list

Navodi sve autentifikovane dobavljače pohranjene u datoteci vjerodajnica.

```bash
cody auth list
```

Ili kratka verzija.

```bash
cody auth ls
```

---

#### logout

Odjavljuje vas s provajdera tako što ga briše iz datoteke vjerodajnica.

```bash
cody auth logout
```

#### install

Ovo postavlja neophodni tok rada GitHub Actions i vodi vas kroz proces konfiguracije. [Saznajte više](/docs/github).

### mcp

Upravljajte Model Context Protocol (MCP) serverima.

```bash
cody mcp [command]
```

---

#### add

Dodajte MCP server svojoj konfiguraciji.

```bash
cody mcp add
```

Ova naredba će vas voditi kroz dodavanje lokalnog ili udaljenog MCP servera.

---

#### list

Navedite sve konfigurirane MCP servere i njihov status veze.

```bash
cody mcp list
```

Ili koristite kratku verziju.

```bash
cody mcp ls
```

---

#### auth

Autentifikujte se sa MCP serverom koji je omogućen za OAuth.

```bash
cody mcp auth [name]
```

Ako ne navedete ime servera, od vas će biti zatraženo da izaberete neki od dostupnih servera koji podržavaju OAuth.
Također možete navesti servere koji podržavaju OAuth i njihov status autentifikacije.

```bash
cody mcp auth list
```

Ili koristite kratku verziju.

```bash
cody mcp auth ls
```

---

#### logout

Uklonite OAuth vjerodajnice za MCP server.

```bash
cody mcp logout [name]
```

---

#### debug

Otklanjanje grešaka (debug) OAuth veze sa MCP serverom.

```bash
cody mcp debug <name>
```

### models

Navedite sve dostupne modele konfiguriranih provajdera.

```bash
cody models [provider]
```

Ova naredba prikazuje sve modele dostupne kod vaših konfiguriranih provajdera u formatu `provider/model`.
Ovo je korisno za pronalaženje tačnog naziva modela za korištenje u [vašoj konfiguraciji](/docs/config/).
Opciono možete proslijediti ID provajdera za filtriranje modela po tom dobavljaču.

```bash
cody models anthropic
```

#### Opcije

| Opcija                                  | Opis                                                                     |
| --------------------------------------- | ------------------------------------------------------------------------ |
| <nobr><code>{"--refresh"}</code></nobr> | Osvježite keš modela sa models.dev                                       |
| <nobr><code>{"--verbose"}</code></nobr> | Koristite detaljniji izlaz modela (uključuje metapodatke poput troškova) |

Koristite `--refresh` zastavicu da ažurirate keširanu listu modela. Ovo je korisno kada su novi modeli dodani provajderu i želite da ih vidite u Cody.

```bash
cody models --refresh
```

### run

Pokrenite Cody u neinteraktivnom modu tako što ćete direktno proslijediti prompt.

```bash
cody run [message..]
```

Ovo je korisno za skriptiranje, automatizaciju ili kada želite brz odgovor bez pokretanja punog TUI-ja. Na primjer:

```bash "cody run"
cody run Explain the use of context in Go
```

Također možete priključiti pokrenutu `cody serve` instancu kako biste izbjegli vrijeme hladnog pokretanja MCP servera pri svakom pokretanju:

```bash
# Start a headless server in one terminal
cody serve

# In another terminal, run commands that attach to it
cody run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Opcije

| Opcija                                   | Kratko | Opis                                                                                          |
| ---------------------------------------- | ------ | --------------------------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |        | Naredba za pokretanje, koristite poruku za argumente                                          |
| <nobr><code>{"--continue"}</code></nobr> | `-c`   | Nastavite posljednju sesiju                                                                   |
| <nobr><code>{"--session"}</code></nobr>  | `-s`   | ID sesije za nastavak                                                                         |
| <nobr><code>{"--fork"}</code></nobr>     |        | Forkujte sesiju pri nastavku (koristiti sa `--continue` ili `--session`)                      |
| <nobr><code>{"--share"}</code></nobr>    |        | Podijelite sesiju                                                                             |
| <nobr><code>{"--model"}</code></nobr>    | `-m`   | Model za korištenje u obliku provider/model                                                   |
| <nobr><code>{"--agent"}</code></nobr>    |        | Agent za korištenje                                                                           |
| <nobr><code>{"--file"}</code></nobr>     | `-f`   | Fajlovi koje treba priložiti poruci                                                           |
| <nobr><code>{"--format"}</code></nobr>   |        | Format: default (formatiran) ili json (sirovi JSON događaji)                                  |
| <nobr><code>{"--title"}</code></nobr>    |        | Naslov sesije (koristi skraćeni prompt ako nije navedena vrijednost)                          |
| <nobr><code>{"--attach"}</code></nobr>   |        | Priključite na pokrenuti Cody server (npr. http://localhost:4096)                         |
| <nobr><code>{"--password"}</code></nobr> | `-p`   | Lozinka za osnovnu autentifikaciju (zadano: `CODY_SERVER_PASSWORD`)                       |
| <nobr><code>{"--username"}</code></nobr> | `-u`   | Korisničko ime za osnovnu autentifikaciju (zadano: `CODY_SERVER_USERNAME` ili `cody`) |
| <nobr><code>{"--dir"}</code></nobr>      |        | Direktorij za pokretanje, ili putanja na udaljenom serveru pri spajanju                       |
| <nobr><code>{"--variant"}</code></nobr>  |        | Varijanta modela (napor zaključivanja specifičan za provajdera)                               |
| <nobr><code>{"--thinking"}</code></nobr> |        | Prikaži blokove razmišljanja                                                                  |
| <nobr><code>{"--port"}</code></nobr>     |        | Port za lokalni server (zadano na nasumični port)                                             |

### serve

Pokrenite Cody headless server za API pristup. Pogledajte [server docs](/docs/server) za kompletan HTTP interfejs.

```bash
cody serve
```

Ovo pokreće HTTP server koji pruža API pristup funkcionalnosti Cody-a bez TUI interfejsa. Postavite `CODY_SERVER_PASSWORD` da omogućite HTTP osnovnu auth (korisničko ime je zadano na `cody`).

#### Opcije

| Opcija                                   | Opis                                                  |
| ---------------------------------------- | ----------------------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | Port na kojem treba slušati                           |
| <nobr><code>{"--hostname"}</code></nobr> | Hostname na kojem treba slušati                       |
| <nobr><code>{"--mdns"}</code></nobr>     | Omogući mDNS otkrivanje                               |
| <nobr><code>{"--cors"}</code></nobr>     | Dodatni origin(i) pretraživača koji dozvoljavaju CORS |

### stats

```bash

```

#### Opcije

| Opcija                                  | Opis                                                                                                       |
| --------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| <nobr><code>{"--days"}</code></nobr>    | Prikaži statistiku za zadnjih N dana (sva vremena)                                                         |

### export

```bash
cody export [sessionID]
```

### import

Uvezite podatke sesije iz JSON datoteke ili Cody dijeljenog URL-a.

```bash

```

Možete uvesti iz lokalne datoteke ili Cody dijeljenog URL-a.

```bash
cody import session.json
cody import https://opncd.ai/s/abc123
```

### upgrade

Ažurira Cody na najnoviju verziju ili određenu verziju.

```bash
cody upgrade [target]
```

Za nadogradnju na najnoviju verziju.

```bash
cody upgrade
```

Za nadogradnju na određenu verziju.

```bash
cody upgrade v0.1.48
```

#### Opcije

| Opcija                                 | Kratko | Opis                                                    |
| -------------------------------------- | ------ | ------------------------------------------------------- |
| <nobr><code>{"--method"}</code></nobr> | `-m`   | Korišteni način instalacije; curl, npm, pnpm, bun, brew |
