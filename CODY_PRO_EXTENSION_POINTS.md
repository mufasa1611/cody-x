# Cody Pro Extension Points

This fork uses opencode as its runtime foundation and adds Cody behavior around the supported local extension surfaces.

## Active Extension Points

### Agent Definitions

Location:

```text
<repo>\.opencode\agent\*.md
```

Use this for Cody routing, safety policy, agent purpose, and default permissions. These files are loaded by the agent config loader and do not require source patches.

Current Cody agents:

- `operator`
- `infra-audit`
- `windows-admin`
- `ssh-operator`
- `docker-operator`
- `systemd-operator`
- `proxmox-operator`
- `backup-operator`
- `web-research`

### Local Tools

Location:

```text
<repo>\.opencode\tool\*.ts
```

Use this for Cody-specific capabilities that can be loaded as tools. Tools should be narrow, structured, and explicit about read-only versus mutation behavior.

Current Cody tools:

- `cody-windows-inspect`
- `cody-ssh-inspect`
- `cody-docker-inspect`
- `cody-systemd-inspect`
- `cody-proxmox-inspect`
- `cody-backup-inventory`

### Project Config

Location:

```text
<repo>\.opencode\opencode.jsonc
```

Use this for local defaults, disabled upstream project tools, provider config, permissions, and Cody's default agent.

### Generated Local Model Config

Location:

```text
<repo>\.opencode\generated\opencode.jsonc
```

Generated by:

```text
<repo>\script\discover-local-models.ps1
```

Use this for first-start Ollama and GGUF model discovery. This file is generated and ignored by git.

### Launcher

Locations:

```text
<repo>\cody-pro.cmd
%APPDATA%\npm\cody-pro.ps1
%APPDATA%\npm\cody-pro.cmd
```

Use this for Cody startup behavior before control reaches the Cody Pro CLI, including local model discovery and global command routing.

## Avoid For Now

- Forking core session logic unless an extension surface cannot support the Cody behavior.
- Adding mutation tools before read-only inspection has test coverage.
- Editing generated model config by hand.
